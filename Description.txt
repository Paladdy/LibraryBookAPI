----------------------------------
Раздел: метаданные

Request headers (заголовки запроса) — это метаданные, которые отправляются клиентом
(например, браузером, приложением или API-клиентом) на сервер вместе с HTTP-запросом.
Они предоставляют серверу дополнительную информацию о запросе,
таких как формат данных, авторизация, параметры сессии и другое.

Используем функцию get_headers чтобы уточнить у клиента, какие именно
данные он хочет получить от сервера.
----------------------------------
Асинхронность: await

Позволяет не ждать выполнения всей программы, перейти
к выполнению других функций-программ, а к этой перейти когда получит ответ.

----------------------------------

Хранение: данные

Данные, в которые были внесены изменения на сервере
хранятся в оперативке RAM - временное хранилище
откуда после перезапуска удаляются.

Чтобы данные не удалялись, нужно записать либо на HDD (Hard Disk Drive)
либо на SSD. SSD быстрее , за счет мгновенного доступа к данным
и отсутствия вращающихся пластин.

Раздел: Ошибки

422 Unprocessable Content

{
    "detail": [
        {
            "type": "missing",
            "loc": [
                "query",
                "name"
            ],
            "msg": "Field required",
            "input": null
        }
    ]
}

Что означает

Структура ошибки:
detail:

Это массив с деталями ошибки.
В данном случае он содержит один объект с описанием проблемы.
type: "missing":

Тип ошибки — "missing", что означает, что отсутствует обязательное поле.
loc: ["query", "name"]:

Указывает, где именно произошла ошибка.
В данном случае:
"query": проблема в параметре, который должен быть передан в строке запроса (query parameters).
"name": имя параметра, который обязателен, но не был указан.
msg: "Field required":

Сообщение об ошибке, которое говорит: "Поле обязательно" (но оно не передано).
input: null:

Указывает, что значение для обязательного параметра было null (или вовсе не передано).
